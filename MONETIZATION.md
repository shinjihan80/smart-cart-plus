# NEMOA 수익화 & Pro 전환 로드맵

## 현재 상태 (v1.5, 베이직 무료 출시)

- 전체 기능 무료 개방, 로컬 저장, 광고 없음
- AI 호출만 일일 한도 (vision 10 · parser 20 · nutrition 5 · url 5)
- Phase 7 파트너 9개 전부 `enabled: false` (스텁)
- 서버 인프라: Next.js SSR + Anthropic API (사용자당 월 평균 비용만 발생)

## Pro 전환 임계점 (Trigger Thresholds)

| 지표 | 베이직 유지 | Pro 출시 준비 | Pro 필수 |
|------|-----|-----|-----|
| **MAU (월간 활성 사용자)** | < 1,000 | 1,000 ~ 5,000 | 5,000 + |
| **DAU (일간 활성 사용자)** | < 300 | 300 ~ 1,500 | 1,500 + |
| **AI 일일 호출 총량** | < 5,000/일 | 5,000 ~ 20,000 | 20,000 + |
| **월 Anthropic 비용** | < ₩50,000 | ₩50k ~ ₩300k | ₩300k + |
| **localStorage 한도 초과 문의** | 0 | 월 5+ | 월 30+ |
| **기기 간 동기화 문의** | 0 | 월 10+ | 월 50+ |

### 각 지표 의미

**MAU 1,000 도달 시**
- 퍼블릭 PMF(Product-Market Fit) 신호
- 이 시점에 Pro 출시 준비 본격화 (결제 시스템·서버 구축)

**MAU 5,000 도달 시**
- 더 미루면 베이직 AI 비용이 월 30만원+ 돼서 개인 부담 한계
- Pro 출시 필수 — 전환율 3%만 잡아도 손익 분기

**AI 호출 5,000/일**
- 사용자 500명이 하루 10회씩 쓰는 수준
- 이 구간부터 AI 비용이 실질적 고정비가 됨

## Pro 전환 단계 (3-Phase Rollout)

### Phase A: 결제 인프라 (MAU 1,000 시점)

**구축 작업**
- Supabase 프로젝트 셋업 (인증만, 데이터는 여전히 로컬)
- 토스페이먼츠 빌링 연동 (월 정기결제 ₩4,900)
- Next.js API 라우트에 Pro 판별 미들웨어

**기간**: 2-3주
**비용**: Supabase Hobby $0/월, 토스 결제 수수료 3.3%

### Phase B: Pro 기능 차등화 (MAU 3,000 시점)

**베이직 (무료 유지)**
- 전체 UI, 로컬 저장, 백업 JSON
- AI 호출 일일 한도 (현재 값 유지)
- 저장 코디 20개, 프로필 2명
- Phase 7 파트너 **비활성**

**Pro (₩4,900/월 또는 연 ₩49,000)**
- ⭐ AI 무제한 (한도 제거)
- ⭐ 클라우드 동기화 — 기기 간 자동 백업 (Supabase 저장)
- ⭐ 저장 코디·프로필 무제한
- ⭐ Phase 7 파트너 전체 활성화 (쿠팡·마켓컬리·당근 등 9개 제휴)
- ⭐ 프리미엄 레시피 100종+ 추가
- ⭐ 광고 제거 (베이직에 배너 도입할 수도)

**기간**: 3-4주
**예상 전환율**: 무료 사용자 대비 2-5% (카테고리 평균)

### Phase C: 제휴 수수료 활성화 (MAU 5,000 시점)

**파트너 연결**
- 쿠팡파트너스 API 검토 + 딥링크 삽입
- 당근마켓 B2B 연동 문의
- 아름다운가게·세탁특공대 제휴 미팅

**수익 구조**
- Pro 구독: 전환율 3% × MAU 5,000 = 150명 × ₩4,900 = **월 ₩735,000**
- 파트너 수수료: 쿠팡 3-5%, 당근 건당 수수료
- 광고 (베이직만): CPM 기준 ₩300,000/월

**기간**: 4-6주 (파트너 계약 리드타임 포함)

## 베이직 단계에서 해두면 좋은 것

현 시점에서 **Pro 전환을 위한 토대는 이미 마련되어 있음**:

- ✅ `partnerLinks.ts` 레지스트리 (enabled:true만 바꾸면 활성화)

- ✅ `profile.ts` 프로필 수 무제한 (UI에서만 제한 걸면 됨)

2. **Sentry 에러 리포팅** — Pro 출시 전에 버그 잡기
3. **홈에 "Pro는 곧" 배너** — 사전 등록 받기 (이메일 수집)

## 결론

**현 v1.5 베이직은 "1,000명까지 운영 가능한 제품"**입니다.
그 이상부터는 AI 비용이 빠르게 증가해 Pro 도입이 강제됩니다.

**추천 릴리즈 순서**:
1. 지금 → 베이직 출시 (무료, 로컬 저장)
2. MAU 1,000 → Phase A (결제 인프라)
3. MAU 3,000 → Phase B (Pro 기능 차등화)
4. MAU 5,000 → Phase C (파트너 수수료)

위 단계 각각 2-4주 리드타임이 필요하므로, MAU 지표가 선형 증가한다면
**출시 후 3-6개월 내 Pro 출시**가 현실적인 타임라인입니다.
