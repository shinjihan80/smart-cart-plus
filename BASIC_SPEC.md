## 한 줄 포지션

> "네모 속에 일상을 모으다 — 냉장고와 옷장을 동시에 관리하는 AI 라이프스타일 비서.

## 핵심 가치

| 영역 | 제공 |
|------|------|
| **개인정보** | 모든 데이터 localStorage 저장 — 서버 전송 없음 |

## 포함 기능 (전체 개방)

### 1. 홈 대시보드

- NEMOA 브랜드 헤더 + 시간대별 네모아 인사 + **⌘K 빠른 탐색**
- **전체 검색** — 상품/레시피/제철 재료 동시 매칭
  - ⌘K · `/` · Esc 단축키
  - 제철 재료 매칭 시 "🌸 제철 딸기 냉장고에 담기" 원탭
  - 레시피 매칭 시 📖 레시피 모달 바로
  - 최근 검색어 5개 자동 저장
- **퀵 링크 4개** (제철 · 레시피 · 쇼핑 · 프로필) + 동적 배지
- **제철 퀵 칩** — 현재 계절 재료 원탭 쇼핑 담기
- **네모아의 오늘 한 마디** — 11+종 시그널 큐레이션
  - 식품 임박 · 제철+임박 · 날씨 · 계절 꺼내기 · 영양 부족 · 미도전 즐겨찾기
  - 쇼핑 리스트 알림 · 제철 피크 장보기 · 자주 같이 만든 메뉴 · 미착용 의류
- **제철 힌트 위젯** · **제철 체크리스트** · **데일리 브리핑** (Open-Meteo 실시간)
- **오늘 한 그릇** — Top 매칭 레시피 + 제철/단골 뱃지
- **옷장 현황** · **이번 달 지출** · **냉장고 카루셀** (D-Day + 스와이프)
- **월별 소비** · **주간 인사이트** · **최근 등록** · **오늘의 팁**

### 2. 스마트 냉장고

- 식품 **11종 카테고리** · D-Day 프로그레스 + 영양 + 이미지
- 🌸 제철 뱃지 + D-2 이내 ⚠️ 강화 경고
- 2단 필터(보관·식품 그룹) + 🌸 제철 필터 + 3-way 정렬
- **제철 재료 섹션** + 검색 시 미보유 제철 자동완성
- **이번 주 영양 밸런스** — 4대 영양소 커버리지 + 네모아 조언
- **오늘 뭐 먹지?** — 랜덤 + 다시 고르기, 단골/제철 뱃지
- **레시피 42종** — 6축 가중치(매칭·임박·계절·단골·영양·난이도)
- 전체 보기 모달(7종 필터) + 상세 모달(스텝·타이머·♥·🌸 하이라이트·조리 로그·같이 만든 메뉴)
- **재구매 추천** — 소진 히스토리 기반
- 카드 확장에 📖 이 재료 레시피 버튼

### 3. 스마트 옷장

- 패션 **13종 카테고리** + 실기온 기반 코디 추천
- **매칭 뱃지 4단계** (✨/👍/🌱/🧊)
- **5-way 정렬** — 이름 · 두께 · 오늘 어울림 · 🔥 자주 입는 · 🌙 오래 안 입은
- **계절 꺼내기 배너** — 보관 중 + 현재 계절 맞는 옷 일괄 꺼내기
- **가상 코디 미리보기** — ownership 필터(전체/개별/공용)
- 스와이프 카드 상세 — 👕 오늘 입었어요 · 같이 자주 입는 조합 TOP 3

### 4. 마이페이지

- **백업 상태 배너** — 7일 stale 감지 + "지금 백업" CTA
- 종합 통계 + 보관 현황 + 카테고리 분포
- 최근 소진 · 월별 지출 추이 · 📅 올해 활동 요약
- **🪄 장볼 거 추천** — 4신호(임박·주기·소진·제철) 자동 추천
- **🛒 쇼핑 리스트** — source별 그룹화 + 5개+ 인라인 검색
- **착용 로그 분석** — 자주 입는/오래 안 입은/아직 안 입은 TOP 3
- **조리 로그 분석** — 자주 만든/오래된/미도전 TOP 3 + 요일 패턴
- **🌸 제철 식탁 히스토리** — 올계절 드신 재료 랭킹 + 못 드신 "모두 담기"
- **🌸 계절 보관** · **🧹 옷장 정리 제안** (60일+ 미착용)
- **🚀 파트너 로드맵** — 9개 파트너 **표시만** (전부 `enabled: false` 스텁)
- 아카이브 · 알림 설정 · 설정

### 5. 설정

- **🧠 네모아가 알고 있는 것** — 레시피 42 · 제철 48 · 카테 24 · 파트너 9
- 🌸 **제철 달력** 링크 → `/seasonal`
- **백업** — 지금 백업 · JSON 복원 · JSON/CSV 내보내기 · 🛡️ 백업 후 초기화
- **🧹 검색어·필터 초기화** · **전체 데이터 초기화**
- **🤖 AI 오늘 남은 횟수** 카드 (아래 "AI 한도" 참고)
- **🩺 오류 기록** 카드 (아래 "에러 로깅" 참고)
- **프로필 관리** — 최대 2명
- **알림 설정** · **앱 정보** · **⌘K 팔레트 버튼**

### 6. `/seasonal` 제철 달력

- 4계절 × **48종** 식재료 풀 카탈로그
- 계절별 / 월별 뷰 토글 (12개월 타임라인)
- 재료 검색 + 피크 우선 정렬
- URL 쿼리 동기화 (`?view=month&season=가을`)
- 쇼핑 담기 + 📖 레시피 바로

### 7. ⌘K 명령 팔레트 (전역)

- 단축키 — ⌘K/Ctrl+K, `/`, Esc
- 검색 범위 — 레시피 8 · 제철 6 · 보유 아이템 5 · 페이지 8 · 액션 5
- Fuzzy 매칭 + Prefix 모드(`>` `#` `?`)
- 최근 실행 5개 LRU + 빈 상태 추천 칩

### 8. 상품 등록 (4가지)

| 방법 | AI | 비고 |
|------|----|----- |
| 사진 분석 | Vision | 자동 분류 + weatherTags |
| URL 분석 | URL agent | og:image 자동 첨부 |
| 빠른 추가 | — | Unsplash 자동 이미지 |
| 텍스트 입력 | Parser | 영수증/이메일 파싱 |

### 9. 개인화 데이터

- 의류 착용 로그 (`nemoa-wear-log`)
- 레시피 조리 로그 (`nemoa-cook-log`) — 3회+ 단골 뱃지
- 즐겨찾기 · 쇼핑 리스트 · 최근 검색어 · 필터/정렬/접기 상태
- **모두 백업 JSON v3에 포함** → 새 기기 수동 복원 가능

## 한도 · 제약 (베이직 규정)

| 항목 | 한도 | 근거 |
|------|------|------|
| **AI vision** (사진 분석) | 10회/일 | Anthropic 비용 · 자정 리셋 |
| **AI parser** (텍스트 파싱) | 20회/일 | 동상 |
| **AI nutrition** (영양 분석) | 5회/일 | 동상 |
| **AI url** (URL 분석) | 5회/일 | 동상 |
| **저장 코디** (`savedOutfits`) | 최대 20개 | localStorage 용량 관리 |
| **프로필** | 2명 | 가족 최소 단위 |
| **레시피 즐겨찾기** | 무제한 | — |
| **아이템 수** | 무제한 (localStorage 약 5MB 한도 내) | — |
| **파트너 연동** | 모두 스텁 (`enabled: false`) | Pro 출시 시 활성 |
| **클라우드 동기화** | 없음 | 수동 JSON 백업만 |
| **광고** | 없음 | — |

AI 한도 도달 시 `TextImportModal`에서 사용자에게 "자정 이후 다시 이용 가능" 안내.

## 운영 인프라 (베이직)

- **배포**: Vercel / Netlify (Next.js SSR) — 서버리스 함수로 `/api/agents/*`
- **비용**: Anthropic API 호출비만 발생 (사용자당 한도 내)
- **모니터링**: 없음 (Pro에서 Sentry·PostHog 도입 예정)
- **결제**: 없음

## 사용자 여정 (베이직)

1. **첫 방문** → `ConsentGate` 동의 모달 → "빈 상태" or "샘플 22개"
2. **상품 등록** → 사진/URL/빠른/텍스트 4가지 중 선택
3. **일상 사용** → 홈 위젯, 냉장고/옷장 관리, 레시피 추천
4. **데이터 보관** → 7일마다 백업 배너 → JSON 수동 다운로드
5. **AI 한도 도달** → 다음 날 00시 리셋 대기
6. **문의** → 설정 > 오류 기록 > 복사 → 이메일 첨부

## "베이직으로 충분한가?" 체크리스트

다음을 만족하면 Pro 없이도 불편 없음:

- ✅ 한 기기만 사용 (여러 기기 간 동기화 불필요)
- ✅ 하루 AI 호출 10-20회 미만
- ✅ 저장 코디 20개 이하로 관리
- ✅ 가족 2명 이하 프로필
- ✅ 파트너 연동(쿠팡·마켓컬리 등) 불필요

위 중 하나라도 한계에 부딪히면 → [PRO_SPEC.md](./PRO_SPEC.md) 참조.
